本笔记为本人寒假期间学习python所记，基于廖雪峰所写Python教程。

数据类型

list

tuple

An unchangeable list

dict

Python内置字典，相当于在其它语言中的map，用key-value存储，查找速度快，用空间换取时间。

**Warning：**由于dict通过key计算value的存储位置，dict的key就必须为不可变对象。而list是可变的，无法作为key。

set

一组key的集合，但不存储value。可以看作数学意义上的无序和无重复元素的集合。

Warning: set原理同dict，故也不可放入可变对象（无法判断可变对象是否相等以保证set中无重复元素）

Summary

不可变对象：str , dict , set , tuple

对于不可变对象来说，调用对象自身的任意方法，也不会改变该对象自身的内容。
相反，这些方法会创建新的对象并返回，这样，就保证了不可变对象本身永远是不可变的。

补充

关于dict的理解——哈希表的相关知识

  1. list和tuple其实是用链表顺序存储的，也就是前一个元素中存储了下一个元素的位置，故list的名字其实是指针，指向list的第一个元素的位置。list的插入和删除等操作可直接用链表的方式进行。

    E.G. 为了在1st和2nd中插入一个元素，直接在链表最后(3rd)加上，然后把第一个元素指针指向这个元素，新元素指针指向原来的第二个元素。读取这个list时，先用list的名字（一个指向第一个元素位置的指针）找到1st的位置，然后用第一个元素的指针找到第二个元素，以此类推。所以list的顺序和内存中的实际顺序其实不一定完全对应。这种存储方式不会浪费内存，但查找起来特别费时间。

  2. dict则采用哈希表。哈希表采用哈希函数从key计算得到一个数字，然后直接把value存储到这个数字对应的地址。

    E.G. key=‘ABC’,value=10 经过哈希函数算得值为123，那就申请有1000个地址（0-999）的内存，然后把10存储在地址为123的地方。因此查表只要给出key，算出哈希值，就可以直接找到value，不必遍历。但内存浪费严重。

    所以key要用不可变对象，因为不可变对象是常量，每次的哈希值算出来都是固定的，不会乱套

  3. 对于不同的key，有很大的概率得到的哈希值也不同。那么有很小的概率不同的key可以得到相同的哈希值。比如哈希值只有3位，那么只要元素个数超过1000，就一定会有至少两个key的哈希值相同（鸽笼原理）。这种情况叫哈希冲突，要想办法补救。
